# Facebook 貼文 — Hybride PageIndex RAG

---

🚀 自己動手做了一套「會看文件、又會思考」的私有化 AI 知識庫:Hybride PageIndex RAG

公司裡一堆 PDF —— SOP、規格書、操作手冊、論文,要找一句話常常翻到天荒地老;想丟給雲端 AI 幫忙找,又怕機密文件外流。😩

所以我做了一套 **資料完全留在本地、離線也能跑** 的文件問答系統。它不只是「搜尋」,而是真的會讀、會推理、會引用出處。來看看它能做什麼 👇

📄【上傳就懂】
PDF 丟進去,系統用 MinerU 自動解析 —— 連 **表格、數學公式、圖片說明** 都抓得到,再把文件拆成「章節樹」建立索引。掃描檔、雙欄排版也處理得來。

💬【像 ChatGPT 一樣對話】
聊天介面 **逐字串流** 回答,還會 **記得上下文**(可以直接追問「那它的限制呢?」),左側有歷史對話可隨時切換重看。每則回答都能展開 **「思考過程」**,看到 AI 每一步在做什麼、呼叫了哪些工具。

🧠【會自己找答案的 Agent】
重點來了 —— 它不是死板的關鍵字比對。背後是一個自主代理(Agent),會主動做 **「向量語意 + 關鍵字」混合檢索**,先鎖定最相關的章節,再回資料庫把那幾頁的 **原文** 撈出來,根據真實內容作答。

🔖【每句都可追溯】
答案會附上 **頁碼引用**,點一下直接跳到原文那一頁對照,不怕 AI 唬爛、不怕「一本正經地胡說八道」。

🙋【不會就說不會】
查無資料會老實說「目前查不到」;問題太模糊,它會 **反問你補充條件**,而不是硬編一個答案出來。對企業應用來說,這點超重要。

🧪【附 MinerU 解析測試台】
還做了一個互動測試頁,可即時切換 **不同解析引擎、語言、公式/表格開關**,比較同一份 PDF 的解析效果,方便針對難解文件調校。

🔧【技術組成】
FastAPI + React 19 前端 + Agno agent 框架 + Qdrant 向量資料庫 + PostgreSQL + 本地 LLM(LM Studio,跑在 RTX 5090)。**全程地端、資料不出門**,適合對隱私與機密有要求的環境。

🎯【適合誰】
企業 SOP 問答、技術文件查詢、客服知識庫、研究文獻整理 —— 任何「文件多到找不到答案」的場景。

把一整櫃的文件,變成一個隨問隨答、還會附出處的 AI 助理。資料不出門,也能擁有自己的 ChatGPT。
